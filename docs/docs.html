Getting Started

Welcome to RenderAI — a system that generates seamless AI textures from text descriptions and applies them to 3D objects in Unity in real time.

Installation

RenderAI consists of two components: a Python/Flask server and a Unity project. Both must be running simultaneously.

ℹ
Prerequisites: Python 3.10+, Unity 6 (Universal 3D), and an internet connection for the Pollinations AI API.

1 — Flask server setup

bash
pip install flask requests pillow
python servidor.py

The server starts on http://localhost:5000 by default. You should see:

output
 * Running on http://127.0.0.1:5000
 * Debug mode: on

2 — Unity project setup

  1. Open Unity 6 and create a new Universal 3D project
  2. Add the AplicarTextura.cs script to any 3D object in your scene
  3. In the Inspector, fill in the Descricao field with your texture prompt
  4. Enable Gerar Ao Iniciar to generate automatically on Play, or call GerarTextura() manually
  5. Press Play — the texture will appear on the object within ~1 second

Quick start

With both the Flask server and Unity running, add the script to a cube and set a description:

Example descriptions
stone wall
wood planks
grass field top view seamless
rusted metal plate
cracked concrete

RenderAI automatically appends , seamless texture, high quality, 4k, tileable to every prompt before sending it to the AI. You don't need to include these manually.

✔
Tip: For best results, describe the material, not the scene. "brick wall" works better than "a wall in a building". See Prompt basics for more examples.

System architecture

RenderAI follows a 3-layer architecture:

Unity / C#
User writes a description in the Inspector or GUI. The AplicarTextura.cs script sends an HTTP POST with the prompt and parameters to the local Flask server.
↓ HTTP POST (JSON)
Flask Server / Python
Receives the request, enriches the prompt, builds a Pollinations AI URL with width, height, and seed parameters, fetches the image, resizes to target resolution, converts to RGB, and returns PNG bytes.
↓ GET image.pollinations.ai
Pollinations AI
Free, no-quota image generation API powered by Stable Diffusion. Responds in avg 0.7s with a JPEG/PNG image at the requested resolution.

Flask server

The server (servidor.py) exposes a single POST endpoint at /gerar-textura.

Request body

JSON
{
  "descricao": "stone wall",
  "resolucao": 1024,
  "seed": 42,
  "steps": 30,
  "guidance_scale": 7.5
}

Response

Returns a image/png binary stream — the generated texture ready to load as a Unity Texture2D.

Prompt enrichment

The server appends quality keywords automatically unless they're already present:

Python
keywords = ", seamless texture, high quality, 4k, tileable"
if "seamless" not in descricao.lower():
    descricao_enriquecida = descricao + keywords

Error handling

The server includes a global try/except with detailed logging, a 120-second timeout, and descriptive error messages returned to Unity if generation fails.

Unity C# script — AplicarTextura.cs

The script runs on any GameObject with a Renderer component. It uses Unity Coroutines to fetch and apply textures without blocking the main render thread.

Inspector fields

FieldTypeDescription
DescricaostringTexture prompt (e.g. "stone wall")
ResolucaointOutput resolution in px (default: 1024)
Gerar Ao IniciarboolGenerate automatically on Play
SeedintRandomness seed (same seed → same texture)

Key implementation details

  • TextureWrapMode.Repeat — ensures seamless tiling on all surfaces
  • FilterMode.Bilinear — smooth texture at all camera angles
  • estaCarregando flag — prevents duplicate requests during generation
  • 120s timeout — avoids Unity hanging if the server is unreachable
  • JsonUtility serialization — safe handling of special characters in prompts
C# — calling from code
// Get the component and generate a new texture at runtime
AplicarTextura at = cube.GetComponent<AplicarTextura>();
at.descricao = "rusted iron plate";
at.GerarTextura();

Prompt basics

RenderAI is optimised for material/texture descriptions — not scene descriptions. Think of what the surface looks like, not what object it's on.

Good prompts ✔Less effective ✗
stone walla building with stone
wood planks, dark oaka wooden floor in a house
grass ground top viewa garden scene
cracked concrete, weathereda broken pavement
rusted metal plate, orangean old car door
⚠
Top-view textures: For floor textures (grass, dirt, sand), add "top view" to your prompt. Without it, the AI may generate a landscape with sky instead of a seamless surface.

✗ "grass field" → generates a landscape
✔ "grass ground top view seamless" → generates a tileable texture

Automatic prompt enrichment

RenderAI automatically adds quality keywords to every prompt to improve output consistency. This happens server-side before the request is sent to Pollinations AI.

Input promptSent to AI

FAQ

Do I need a GPU?

No. RenderAI uses the Pollinations AI cloud API for texture generation — no local GPU is required. Any modern computer can run it.

Why did the original Hugging Face API get replaced?

The Hugging Face free tier had significant response time variability (10–60 seconds) and endpoint instability. Pollinations AI provides free, no-quota access with average latency under 1 second.

Can I use the same texture repeatedly?

Yes — use the same seed value and prompt. The same seed + prompt always produces the same texture, making generation reproducible.

Why does my grass texture look like a landscape?

Add "top view" to your prompt: "grass ground top view seamless". Vague prompts like "grass field" may generate a scene with sky and horizon.

Can I use RenderAI in a commercial project?

RenderAI itself is an academic project (UTAD, 2026). Textures generated via Pollinations AI are subject to their terms of service. Always verify licensing for production use.

Does it work offline?

No. Texture generation requires an internet connection to reach the Pollinations AI API. The Flask server and Unity rendering work locally.
